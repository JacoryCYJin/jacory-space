# 示例功能 PRD

## 目标

用一句话说明这个功能要解决什么问题，或者最终希望页面/工具变成什么状态。

示例：让 Blog 列表更像个人档案索引，方便快速浏览文章。

## 范围

写清楚这次做什么、不做什么，避免开发时范围越变越大。

做：

- 调整文章列表的信息层级
- 显示标题、日期、分类、摘要
- 保持现有路由不变

不做：

- 不接 CMS
- 不改文章详情页
- 不新增搜索

## 行为

写用户能看到什么、能操作什么、操作后发生什么。

- 页面显示文章列表
- 每篇文章显示标题、日期、分类和摘要
- 点击文章后进入对应详情页
- 没有文章时显示简短空状态

## 数据

写这个功能需要哪些字段、接口、配置或本地数据。

每篇文章需要：

- `title`
- `date`
- `category`
- `summary`
- `slug`

## 验收

写做到哪些点就算完成。

- Blog 页面能正常显示文章列表
- 点击文章能进入详情页
- 移动端内容不溢出
- 页面风格符合 Personal Operating System 视觉规则
- `npm run build` 通过
